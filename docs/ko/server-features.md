# 이 서버 전용 기능

<img src="../assets/player-guides/openmoji/server-features.svg" alt="데스크톱 컴퓨터 아이콘" width="96">

이 문서는 친구가 실제로 사용하는 기능을 한곳에 모은 빠른 안내입니다. 설정·운영
세부사항은 링크된 관리자 문서를 참고하세요. Discord 명령 이름은 사용자의 Discord 앱
언어에 따라 `/서버` 또는 `/server`처럼 표시될 수 있습니다.

## 처음 접속하기

1. 서버장이 Discord `/관리자` → **친구 계정**에서 친구를 선택합니다.
2. 정확한 Java 닉네임 또는 Bedrock/Xbox 게이머태그를 추가합니다. 한 Discord 사용자에
   여러 Minecraft 계정을 등록할 수 있습니다.
3. 봇이 Java 화이트리스트 또는 Floodgate 화이트리스트를 함께 갱신합니다.
4. 친구는 `/서버`에서 주소·버전·현재 접속자와 플레이 일수를 확인하고 접속합니다.

계정 추가·삭제는 서버 상태를 바꾸므로 관리자만 할 수 있습니다. 친구가 임의의
Minecraft 닉네임, 좌표, RCON 명령을 입력해 다른 사람을 움직이는 방식은 제공하지
않습니다.

## 공개 명령

| 명령 | 공개 범위 | 용도 |
|---|---|---|
| `/서버` | Discord 서버 구성원 | 접속 주소, 버전, 접속자, 플레이 일수, 새로고침 |
| `/도구` | 등록된 친구·관리자 | 내 계정, 구조·위치, 좌표북, 일지, 데스박스, 서버 점수 |
| `/도움말` | Discord 서버 구성원 | 관리자 기능을 숨긴 친구용 도움말 |
| `/업로드 좌표사진` | 등록된 친구·관리자 | 좌표 항목의 사진 첨부/교체 |
| `/업로드 일지` | 등록된 친구·관리자 | 사진이 포함된 일지 작성 |

응답 중 계정·좌표·관리 정보가 있는 패널은 실행한 사람에게만 보이는 비공개 응답을
사용합니다.

## 내 계정과 구조

`/도구`에서 관리자가 연결해 둔 Minecraft 계정을 먼저 선택합니다.

- **내 계정 목록:** Java·Bedrock 계정과 현재 선택을 확인합니다.
- **선택 계정 위치:** 그 계정이 온라인일 때 현재 차원과 XYZ를 읽습니다.
- **선택 계정 스폰 귀환:** 그 계정이 온라인일 때 서버의 실제 월드 스폰으로만
  이동합니다. 좌표나 다른 대상을 입력할 수 없습니다.

스폰 귀환은 제한된 `raspiops rescue` 플러그인 명령을 사용합니다. 친구에게 op나
일반 텔레포트 권한을 주지 않습니다.

## 좌표북

- 연결된 온라인 계정의 현재 위치를 자동으로 읽어 이름·메모와 함께 저장합니다.
- 오버월드·네더·엔드 차원과 XYZ를 보관하며, 외부 웹 지도가 설정된 경우 링크도
  표시합니다.
- PNG/JPEG/WebP/GIF 사진을 5MiB 이하로 첨부할 수 있습니다. Discord의 만료 URL이
  아니라 서버 HDD에 한 번 저장합니다.
- 친구는 자신이 만든 좌표만 삭제하고 관리자는 모두 관리할 수 있습니다.
- 전체 좌표는 Pi의 파일 읽기와 메뉴 크기를 제한하려고 250개까지 저장합니다.

광산 입구, 네더 포털, 마을, 농장, 시련의 회당처럼 다시 방문할 장소는 죽기 전에
좌표북에 남겨 두세요.

## 서버 일지

- `/도구` → **서버 일지**에서 최근 기록을 보고 새 글을 씁니다.
- 사진 없는 글은 버튼과 모달로, 사진이 있는 글은 `/업로드 일지`로 작성합니다.
- 구조와 좌표 저장 같은 주요 이벤트도 자동으로 기록됩니다.
- 일지는 추가 전용 JSONL이며 2MiB를 넘으면 최신 1,000개를 보존하도록 정리합니다.

일지에는 공개해도 되는 모험 기록만 쓰세요. Discord 사용자 ID, Minecraft 계정,
사진과 글은 공개 Git 저장소가 아니라 운영 HDD에 저장되지만 서버 관리자는 볼 수
있습니다.

## Death Box

일반 사망에서 `keepInventory`가 꺼져 있고 드롭이 있으면 Death Box 플러그인이 아이템을
바닥에 뿌리는 대신 보호 상자에 보관합니다.

- 사망자와 관리자만 기본적으로 열 수 있습니다.
- 폭발, 피스톤, 호퍼, 권한 없는 파괴·열기를 막습니다.
- 안전한 물리 상자 위치가 없으면 아이템을 가상 상자에 보관합니다.
- `/도구`의 **데스박스 찾기/목록**으로 연결된 내 계정의 상자를 조회합니다.
- 상자를 열어 완전히 비우면 기록과 물리 상자가 정리됩니다.

`keepInventory=true`이면 Death Box를 만들지 않습니다. 관리자 설정과 복구 명령은
[Death Box 설계](death-box-design.md)를 참고하세요.

## 스폰 보호와 상자 잠금

- **스폰 보호:** 기본적으로 주 월드 스폰 중심 16블록 정사각형에서 일반 사용자의
  설치·파괴·상호작용·양동이·엔티티 공격·폭발 피해 등을 막습니다. OP는 우회합니다.
- **상자 잠금:** 설치자를 기억해 상자·덫 상자·통·셜커 상자를 다른 플레이어가 열거나
  부수지 못하게 합니다. 호퍼 이동까지 막는 토지 보호 기능은 아니므로 중요한 물건은
  통이나 셜커 상자에 두고 자동화 연결을 확인하세요.
- 스폰 안의 본인 Death Box는 회수할 수 있도록 보호 기능끼리 예외 처리됩니다.

두 기능은 관리자가 Discord에서 켜고 끌 수 있으며 상태가 재시작 뒤에도 유지됩니다.
세부 설정은 [Paper 운영 플러그인](paper-ops-plugin.md)을 참고하세요.

## 서버 점수와 성능

`/도구` → **서버 점수**는 버튼을 누른 시점에만 다음 항목을 측정해 0~100점과 감점
이유를 표시합니다.

- Paper TPS와 RCON 응답
- Raspberry Pi CPU 온도·5분 부하
- 메모리 사용률
- HDD 여유 공간과 필수 마운트
- 저전압·스로틀 플래그

점수 확인은 월드 전체 검색이나 상시 폴링을 만들지 않습니다. 점수가 낮으면 자동 농장과
떨어진 아이템을 먼저 점검하고 관리자에게 알리세요.

## 관리자 전용 기능

`/관리자`는 `ADMIN_USER_IDS`에 등록된 사용자만 열 수 있으며 다음 기능을 비공개 버튼
패널로 제공합니다.

- 서버 시작·정지·재시작과 상태 진단
- 접속자 인벤토리·위치·체력·효과 조회
- 제한된 아이템/효과/인챈트/게임 모드/TP/경험치/회복/추방 조작
- 백업 생성·검증·다운로드·복구·정리와 월드 교체
- 시간·날씨·난이도·게임룰·스폰·드롭 아이템 정리
- 친구 계정과 Java/Bedrock 화이트리스트 관리
- 업데이트, 로그, 저장공간, 성능 분석, 스폰 보호·상자 잠금

위험한 정지·재시작·복구·삭제·업데이트·추방·드롭 정리는 확인 버튼을 한 번 더
요구하고 감사 로그를 남깁니다. 전체 운영법은 [Discord 봇](discord-bot.md)과
[운영 런북](operator-runbook.md)을 참고하세요.

## 데이터와 개인정보

친구 계정, 좌표, 일지, 사진은 기본적으로 `/mnt/minecraft/bot-state`에 저장됩니다.
월드와 함께 민감한 사설 서버 데이터로 취급하며 Git에 커밋하지 않습니다. 공개
좌표나 일지에도 실제 집 주소, 전화번호, 토큰 같은 개인정보를 쓰지 마세요.

# 추천 플러그인 (선택 설치)

이 문서는 친구용 야생 서버(라즈베리 파이 4B, 3~4명)에 재미를 더하는 **외부**
Paper 플러그인을 정리합니다. 이 저장소에 번들된 `RaspiMcOps`·`DeathBox`와 달리,
아래 플러그인들은 각자 배포처에서 직접 받아 설치해야 합니다.

> ⚠️ **버전 호환 주의**: 이 서버는 Paper 26.1.x를 씁니다. 아래 플러그인의 공개
> 호환 목록이 항상 최신 마인크래프트까지 명시하지는 않으므로, **먼저 테스트
> 서버(또는 백업 후)에서 확인**한 뒤 실서버에 올리세요. 특히 Lootin은 확인된
> 공개 호환 번호가 구버전까지만 명시된 경우가 있어 26.1.x에서 반드시 사전
> 검증이 필요합니다.

## 설치 방법

다운로드 URL은 플러그인 버전마다 바뀌므로 저장소에 넣지 않습니다. 배포처
(Modrinth·Hangar·SpigotMC 등)에서 **Paper 26.1.x용 최신 JAR의 직접 다운로드
URL**을 복사한 뒤, 안전 설치 스크립트로 서버 `plugins/` 폴더에 넣습니다.

```bash
# URL만 주면 파일명을 유추합니다.
./scripts/install_plugin.sh "<다운로드-URL>"

# 파일명과 체크섬을 지정할 수도 있습니다(권장).
./scripts/install_plugin.sh "<다운로드-URL>" Lootin.jar --sha256 <배포처가-제공한-해시>
```

스크립트는 다운로드 → (선택)체크섬 검증 → 유효한 JAR·plugin.yml 포함 확인 →
`plugins/`로 원자적 이동을 수행하며, 재실행하면 같은 파일을 덮어씁니다(업데이트에
안전). 설치 후 **서버를 재시작**해야 로드됩니다.

수동으로 넣어도 됩니다 — JAR을 `server/plugins/`(운영 서버는 HDD의 `plugins/`)에
복사하고 재시작하세요.

## 우선순위

| 순위 | 플러그인 | 목적 | 파이 부하 |
|---|---|---|---|
| 1 | **Lootin** 또는 **JustLootIt** | 구조물 상자 전리품을 플레이어별로 분리 | 낮음 |
| 2 | **AuraSkills** | 바닐라 기반 스킬 성장(RPG 감성) | 낮음 |
| 3 | **Chunky** | 탐험 렉 방지용 청크 사전 생성 | 생성 중 높음 |

## 1. Lootin / JustLootIt — 상자 전리품 분리 (우선)

친구와 같이 구조물을 발견했을 때, 일반 서버에서는 먼저 연 사람이 다 가져갑니다.
이 계열 플러그인은 **던전 상자를 플레이어별로 한 번씩** 열 수 있게 해서:

- 친구가 먼저 요새를 털어도 내 전리품이 남아 있음
- 사막 사원·폐광·고대 도시·엔드 도시 탐험에 적합
- 상자 때문에 다툴 일이 줄어듦

Lootin은 플레이어 UUID별로 구조물 상자 전리품을 분리합니다. JustLootIt도 같은
방식의 대안입니다. **재미 대비 부하가 낮아 강력 추천**하지만, 위 호환 주의대로
26.1.x에서 먼저 테스트 서버로 확인하세요.

## 2. AuraSkills — 무난한 성장 시스템

기존 야생에 채광·벌목·농사·낚시·전투·궁술·민첩성·방어·마법 계열 능력치와
체력·힘·행운 스탯 성장을 추가합니다. 새 블록/몹을 대량으로 넣지 않고 **바닐라
행동을 기반으로 경험치**를 주며, 베드락(크로스플레이) 플레이어도 참여할 수
있습니다. 기존 월드에 중간부터 설치해도 부담이 적습니다.

**파이 4B 권장 설정**(기본 배율이 다소 빠르게 느껴질 수 있음):

- 전체 경험치 배율: **0.6 ~ 0.8**
- 최대 레벨: 기본값 또는 **50**
- 전투 스킬 보너스 피해량: 약하게
- 체력 증가량: 기본값의 **50%**
- 스킬 메시지·액션바 갱신 빈도: **낮게** (파이 부하·채팅 스팸 절감)

## 3. Chunky — 탐험 렉 방지 (후순위)

지정 범위의 청크를 미리 생성해 탐험 중 실시간 생성 렉을 줄입니다. 랜덤
텔레포트나 월드 생성 기능을 넣을 때 특히 유용합니다.

**파이 4B 권장 범위·운영**:

- 오버월드 반경: **3,000 블록**
- 네더 반경: **1,000 블록**
- 엔드: 기본 상태 유지
- **플레이어가 없을 때** 실행
- **한 월드씩** 순차 실행
- 생성 중 **squaremap 전체 렌더링 동시 실행 금지**(둘 다 저장장치를 강하게 씀)

Chunky는 상시 설치해 두어도 평소엔 거의 일을 하지 않지만, 청크 생성 작업 중에는
CPU와 저장장치를 강하게 사용하므로 위 운영 규칙을 지키세요.

## 설치 후 확인 체크리스트 (실서버에서)

- [ ] 서버 시작 로그에 플러그인이 오류 없이 로드됐는지 (`/plugins` 또는 콘솔)
- [ ] Lootin/JustLootIt: 두 계정으로 같은 구조물 상자를 각각 열어 전리품이 분리되는지
- [ ] AuraSkills: 채광·벌목 등으로 경험치가 오르는지, 배율이 과하지 않은지
- [ ] AuraSkills: 베드락(크로스플레이) 친구도 스킬에 참여되는지
- [ ] Chunky: 아무도 없을 때 한 월드씩 생성하고, squaremap 렌더와 겹치지 않는지
- [ ] 생성 작업 중 TPS·CPU 온도·HDD 여유를 `/관리자` 성능 카드로 모니터링

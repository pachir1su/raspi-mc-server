# Death Box 설계(Paper 플러그인 후속 작업)

Death Box는 의도적으로 **Discord RCON 봇으로 구현하지 않습니다**. RCON은 사망
로그가 도착한 뒤에야 반응할 수 있어, 그사이 드롭 아이템이 용암·폭발·디스폰으로
사라질 수 있습니다. 아이템 메타데이터, 중첩 컨테이너, 인챈트, 소유권도 안전하게
복원하기 어렵습니다.

## 권장 구현 방식

별도 후속 작업에서 작은 Paper 플러그인을 만듭니다. 단순한 서버라면 데이터팩도
가능하지만, 정확한 인벤토리·소유권·보호된 접근·재시작 복구·버전별 테스트까지
고려하면 플러그인이 더 안전합니다.

플러그인은 이벤트 기반으로 동작합니다.

1. Paper의 플레이어 사망 이벤트를 일반 우선순위로 받고, 취소됐거나
   `keepInventory`가 적용된 사망은 무시합니다.
2. 방어구와 보조 손을 포함한 전체 `ItemStack`을 이벤트 안에서 메모리로 복사합니다.
3. 복사한 이벤트 드롭만 비워서 아이템이 위험한 바닥 엔티티로 생성되지 않게 합니다.
4. 사망 블록 주변의 작은 범위만 탐색해 안전한 컨테이너 위치를 찾습니다. 청크나
   월드 전체는 절대 검색하지 않습니다.
5. 54칸 이중 상자 같은 컨테이너를 만들고 아이템을 넣습니다. 공허·용암·보호 구역·
   빈자리 부족 때문에 안전한 블록이 없으면 아이템을 지우지 않고 플러그인 내부의
   가상 인벤토리로 보관한 뒤 플레이어에게 알립니다.
6. Paper의 영속 데이터 API로 사망한 플레이어 UUID와 고유 상자 ID를 기록합니다.
   좌표 알림은 해당 플레이어에게만 보냅니다.
7. 상자가 비면 메타데이터 레코드를 삭제합니다. 선택적 만료 시간은 서버장이
   설정하며 기본값은 비활성화입니다.

이 방식은 사망·상자 접근·제한된 정리 시점에만 작업합니다. RCON 폴링, 로그 파싱,
엔티티 전체 검색, 강제 청크 로딩은 하지 않습니다.

## 소유권과 안전 규칙

- 기본 접근자는 사망한 플레이어와 마인크래프트 관리자(op)뿐입니다.
  `friends-can-open` 설정으로만 친구 접근을 선택적으로 허용합니다.
- Discord 사용자가 입력한 임의 명령은 절대 실행하지 않습니다.
- `ItemStack` 메타데이터를 그대로 보존하고 명령 문자열로 직렬화하지 않습니다.
- 기존 블록, 보호 구역, 월드 경계, 로드되지 않았거나 위험한 위치를 확인한 뒤에만
  컨테이너를 놓습니다.
- `keepInventory`, 묘비 플러그인, 토지 보호 플러그인, 사망 드롭 변경 플러그인과의
  호환 규칙을 명시합니다. 다른 묘비 플러그인이 활성화돼 있으면 아이템 복제를
  피하기 위해 이 플러그인을 비활성화합니다.
- 메타데이터는 원자적으로 기록합니다. 시작 시 알고 있는 상자 레코드만 조정하고
  로드된 모든 청크를 검색하지 않습니다.

## 권장 설정

```yaml
enabled: true
container: double-chest
search-radius: 4
expire-hours: 0        # 0이면 만료하지 않음
friends-can-open: false
fallback-virtual-box: true
```

이 값들은 Discord 봇의 `.env`가 아니라 Paper 플러그인 자체 설정에 둡니다.

## 권장 명령

| 명령 | 권한 | 용도 |
|---|---|---|
| `/deathbox locate` | 상자 소유자 | 자신의 최신 Death Box 좌표 표시. |
| `/deathbox list` | 상자 소유자 | 자신의 활성 상자 목록 표시. |
| `/deathbox recover <id>` | 관리자 | 확인 후 가상 대체 상자 복구. |
| `/deathbox purge <id>` | 관리자 | 명시적 확인 후 상자 삭제. |

Discord에는 나중에 제한된 플러그인 API가 돌려주는 상자 좌표를 **읽기 전용**으로만
노출할 수 있습니다. 생성·아이템 캡처·복구·삭제는 Paper 내부에 유지합니다.

## 후속 작업에서 필요한 검증

- 일반 인벤토리, 방어구, 보조 손, 인챈트, 번들, 셜커 상자.
- 용암, 공허, 폭발, 좁은 동굴, 물, 월드 경계, 보호 구역.
- 27칸 초과 사용과 완전히 가득 찬 인벤토리.
- 사망 직후 재시작과 상자를 연 상태에서 재시작.
- 동시 사망, 연속 사망, 같은 좌표에 여러 상자 생성.
- `keepInventory=true`, 다른 사망 플러그인, 호퍼 접근, 폭발, 권한 없는 플레이어.
- Paper timings/profile 비교로 유휴 틱·청크 로드 부하가 없음을 확인.

정확한 Paper 버전과 보호 플러그인 호환 범위를 정한 뒤, 플러그인 전용 새 PR로
구현하고 배포합니다.

# 친구 도구: 계정 연동, 구조, 좌표북, 일지, 건강 점수

Discord 봇은 관리자가 계정을 등록한 친구에게 범위가 좁은 셀프서비스 기능만 제공합니다.
친구를 마인크래프트 op로 만들지 않으며, 친구가 입력한 임의 RCON 문자열을 실행하지
않습니다.

## 1. Raspberry Pi 설정

Pi에 있는 기존 `.env`를 수정합니다. 실제 값이 든 파일은 커밋하지 마세요.

```bash
cd ~/raspi-mc-server
nano .env
```

다음 값을 추가하거나 수정합니다.

```dotenv
# 친구용 cog 활성화. 기존 관리 명령은 계속 관리자 허용목록으로 보호됨.
PUBLIC_COMMANDS_ENABLED=true

# 런타임 JSON과 업로드 사진을 마운트된 HDD에 저장.
MC_STATE_DIR=/mnt/minecraft/bot-state

# 선택 사항: 외부 웹 지도 URL. 웹 지도를 배포하기 전에는 비워 둠.
MC_MAP_URL_TEMPLATE=https://map.example.com/?world={dimension}&x={x}&y={y}&z={z}
```

> 참고: 과거의 `MC_SPAWN_X/Y/Z/DIMENSION` 좌표 오버라이드는 제거되었습니다.
> 스폰 귀환은 항상 **월드 스폰**을 사용하므로 죽었을 때 리스폰 위치와 항상
> 일치합니다. 운영 `.env`에 이 값이 남아 있다면 지워도 됩니다(무시됩니다).

구조 스폰 위치를 고르는 방법:

1. 스폰으로 삼을 자리에 아무 접속자나 세웁니다(서버장 본인 계정 추천).
2. `/관리자` → **빠른 명령** → **스폰 지정**에서 그 접속자를 선택합니다.
   좌표를 직접 입력할 수도 있습니다.
3. 지정하면 죽었을 때 리스폰(침대 없을 때)과 `/도구`의 스폰 귀환이 모두
   그 지점으로 바뀝니다. 분산 반경(`spawnRadius`)도 0으로 함께 맞춥니다.

`MC_MAP_URL_TEMPLATE`은 실제 배포한 웹 지도의 URL 형식과 맞아야 합니다.
`{dimension}`, `{x}`, `{y}`, `{z}` 자리표시자를 쓸 수 있습니다. 봇은 링크만 만들고
지도를 렌더링하거나 폴링하지 않습니다. 지도 URL 형식이 다르면 브라우저에서 한
좌표를 연 뒤 URL을 복사하고 좌표 값을 해당 자리표시자로 바꾸세요. 웹 지도가 없으면
값을 비워 두면 되며, 지도 링크 없이 좌표북은 그대로 사용할 수 있습니다.

`.env` 저장 후 봇을 재시작하고 로그를 확인합니다.

```bash
sudo systemctl restart mc-discord-bot.service
sudo systemctl status mc-discord-bot.service --no-pager
sudo journalctl -u mc-discord-bot.service -n 100 --no-pager
```

## 2. 친구 계정 등록

서버장은 `/관리자`의 **친구 계정**에서 Discord 사용자를 고릅니다. 그다음
**Java(PC) 추가** 또는 **Bedrock(모바일) 추가**를 누르고 정확한 마크닉만 입력합니다.
친구의 연동 요청이나 승인 대기 절차는 없습니다. Discord 사용자 한 명에게 Java와
Bedrock 계정을 여러 개 등록할 수 있고, 삭제할 때도 선택한 계정 하나만 삭제됩니다.
친구는 `/도구`의 **내 계정 목록**에서 등록 결과를 확인합니다.

등록할 때 Java는 `whitelist add`, Bedrock은 Floodgate의 `fwhitelist add`를 자동
실행하고, 삭제할 때는 맞는 해제 명령을 실행합니다. 서버 변경이므로 이 작업은 계속
`ADMIN_USER_IDS` 뒤에 있습니다. Java 이름은 일반적인 1~16자 형식을 따르고 Bedrock
게이머태그는 공백도 받을 수 있습니다. 명령 삽입에 사용될 문자는 모두 거부합니다.

## 3. 연결된 본인 계정만 구조

- `/도구`에서 계정을 고른 뒤 **선택 계정 스폰 귀환**을 누르면 실행한 Discord 사용자에게 등록된 마크닉만 이동합니다.
  플레이어가 온라인이어야 하며, 접속 중이 아니면 봇이 "플레이어가 게임에 접속
  중이 아닙니다"라고 안내하고 아무것도 실행하지 않습니다. 목적지는 항상
  **월드 스폰**(관리 패널의 스폰 지정으로 변경)이므로 죽었을 때 리스폰 위치와
  같습니다. 친구가 대상·좌표·RCON 명령을 입력할 수 없습니다.
- **선택 계정 위치**는 연결된 본인 온라인 플레이어의 차원과 XYZ만 읽습니다.
- **내 상태**는 본인 캐릭터의 체력·허기·경험치·포션 효과를 읽기 쉬운 형태로
  보여줍니다(읽기 전용).
- **서버 시간**은 현재 플레이 일수와 낮/밤을 보여줍니다(읽기 전용).

### 플레이어에게 이동 (쿨타임)

- **플레이어에게 이동**은 접속 중인 다른 플레이어를 골라 본인 계정을 그 자리로
  순간이동합니다. 친구끼리 신뢰를 전제로 상대 동의 절차는 없지만, **쿨타임**을
  두어 반복 기습 이동을 막습니다.
- 쿨타임은 **디스코드 계정 단위**로 적용됩니다. 계정을 여러 개 연동해도 하나의
  타이머를 공유하므로 계정을 바꿔가며 쿨타임을 회피할 수 없고, 봇을 재시작해도
  파일에 저장되어 유지됩니다.
- 기본값: 플레이어 이동 **30분**(`TP_COOLDOWN_SECONDS`), 스폰 귀환 **5분**
  (`RESCUE_COOLDOWN_SECONDS`). Pi의 운영 `.env`에서 조정하세요(초 단위, 0이면
  비활성화). 스폰 귀환 쿨타임은 이동에 성공했을 때만 시작됩니다(미접속 실패는
  카운트하지 않음).

관리자의 계정 등록이 이 고정 TP 한 가지에 대한 위임 권한입니다. 일반 서버
변경, 인게임 명령어(콘솔 명령 직접 실행), 시작·정지, 백업, 화이트리스트, 빠른 명령,
월드 관리 기능은 모두 계속 `ADMIN_USER_IDS` 뒤에 있습니다.

## 4. 좌표북과 사진

승인된 친구와 관리자는 `/도구`의 **좌표북**을 엽니다. 기존 좌표는 드롭다운으로
선택하고 보기·삭제는 버튼으로 실행합니다. **현재 위치 저장**은 연결된 온라인
플레이어의 XYZ를 자동으로 읽고 짧은 이름과 선택 메모만 받습니다. 사진을 붙이거나
교체할 때만 버튼에서 제공할 수 없는 첨부 기능인 `/업로드 좌표사진`을 사용합니다.

사진은 PNG, JPEG, WebP, GIF 형식과 5MiB 이하만 받습니다. 봇은 사진을 한 번
다운로드해 `MC_STATE_DIR/friend-media`에 보관하므로 만료될 수 있는 Discord 첨부 URL에
의존하지 않습니다. 친구는 자신이 등록한 좌표만 삭제할 수 있고 관리자는 모두 삭제할
수 있습니다. Pi에서 읽기 비용과 Discord 출력을 작게 유지하려고 좌표는 250곳으로
제한합니다.

## 5. 서버 일지

승인된 친구와 관리자는 `/도구`의 **서버 일지**에서 최근 기록을 드롭다운으로
선택하거나 **새 일지**를 누릅니다. 글쓰기가 기능 자체이므로 내용만 모달로 받고,
사진이 필요할 때만 `/업로드 일지`를 사용합니다.

구조와 좌표 저장 이벤트는 자동으로 일지에 추가됩니다. 일지는 쓰기 비용이 작은
추가 전용 JSONL을 사용하고, 2MiB를 넘으면 최신 1,000개만 남기도록 정리합니다.
선택 사진은 같은 5MiB 로컬 저장 정책을 사용합니다.

## 6. 요청할 때만 계산하는 서버 건강 점수

`/도구`의 **서버 점수**는 Paper TPS, RCON 연결, CPU 온도, 5분 부하, 메모리, HDD 여유 공간,
Raspberry Pi 저전압·스로틀 플래그를 한 번 측정합니다. 0~100점, 등급, 모든 감점
이유를 보여줍니다. 명령을 실행할 때만 동작하며 새 백그라운드 루프, 청크 검색,
추가 서버 틱 작업은 없습니다.

## 런타임 파일과 백업

권장 HDD 설정을 사용하면 다음 파일이 `/mnt/minecraft/bot-state` 아래에 생깁니다.

- `player-links.json`
- `places.json`
- `server-diary.jsonl`
- `friend-media/`

Discord 사용자 ID, 마크닉, 좌표, 일지 내용, 사진이 들어 있으므로 다른 사설 서버
데이터처럼 보호하고 호스트 수준 HDD 백업에 이 폴더를 포함하세요. Git에서는
무시됩니다.

## 문제 해결

| 증상 | 확인할 것 |
|---|---|
| 친구에게 기능 비활성화 메시지가 표시됨 | `PUBLIC_COMMANDS_ENABLED=true` 설정 후 봇 재시작. |
| `/도구`에 계정이 안 보임 | 서버장이 `/관리자` → **친구 계정**에서 Discord 사용자를 고르고 Java 또는 Bedrock 계정을 추가. |
| 구조 명령에서 `raspiops`를 모른다고 나옴 | 번들 Paper 운영 플러그인이 포함된 Release를 설치하고 Paper와 봇을 재시작. |
| Bedrock 등록에서 화이트리스트 명령을 모른다고 나옴 | `python -m bot.main --setup`을 다시 실행해 Java+Bedrock을 고르고 두 플러그인 설정 생성 여부 확인. |
| 구조/위치 조회가 플레이어를 못 찾음 | 연결된 정확한 Java/Floodgate 계정으로 현재 접속 중인지 확인. |
| 사진 업로드 실패 | 5MiB 미만 지원 이미지인지, `MC_STATE_DIR` 권한·용량이 충분한지 확인. |
| 지도 링크가 없음 | `MC_MAP_URL_TEMPLATE` 설정. 지도 플러그인이 없으면 실시간 지도 링크도 없음. |
| 지도 링크가 엉뚱한 곳을 엶 | 배포한 지도의 실제 URL 형식에 템플릿을 맞춤. |
| 건강 점수에서 HDD 여유가 0으로 표시됨 | `/mnt/minecraft` 마운트 확인. 필수 마운트가 없으면 비정상으로 판정함. |

### 완전히 새로운 Bedrock 계정의 최초 접속

Floodgate 공식 안내에 따르면 `fwhitelist add <게이머태그>`는 이전에 어떤 Geyser
서버에든 접속한 기록이 있는 계정만 바로 찾을 수 있습니다. 완전히 새 Xbox 계정은
Floodgate가 아직 이름을 몰라 첫 등록이 실패할 수 있습니다.

가능하면 외부 Bedrock 포트포워딩을 열기 전 LAN에서 처리합니다. 정확한 친구가 바로
접속할 준비를 한 짧은 점검 시간에:

```text
whitelist off
# 친구가 Bedrock으로 한 번 접속을 시도해 Floodgate가 계정을 인식하게 함
fwhitelist add 정확한게이머태그
whitelist on
whitelist reload
```

로컬 RCON 또는 `/관리자` → **고급 도구** → **인게임 명령어**로만 실행하고 친구에게 이 권한을 주지 않습니다.
중간 단계가 실패해도 즉시 화이트리스트를 다시 켜고 `whitelist list`로 확인한 다음
관리자 **친구 계정** 패널에서 Bedrock 계정을 다시 추가합니다. 화이트리스트를 끈 상태로 계속 운영하지 마세요.
공식 [Geyser 화이트리스트 FAQ](https://geysermc.org/wiki/geyser/faq/)도 참고합니다.

## 데스박스 버튼

`/도구` 패널에 **데스박스 찾기**와 **데스박스 목록** 버튼이 있습니다. RCON으로
DeathBox 플러그인 명령(`deathbox locate <플레이어>`, `deathbox list <플레이어>`)을
실행하므로 게임 안에서 명령어를 직접 칠 필요가 없습니다.

Death Box는 의도적으로 분리했습니다 — 봇이 아니라 Paper 플러그인입니다. 동작 방식,
빌드·설치 방법, 검증 체크리스트는 [death-box-design.md](death-box-design.md)를
참고하세요.

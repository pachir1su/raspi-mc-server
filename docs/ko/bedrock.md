# 베드락 대안

이 저장소는 **자바 에디션(PaperMC)** 서버를 구현합니다. RCON 기반 원격 관리,
화이트리스트, op 기반 "소유자만 치트" 모델에 가장 잘 맞기 때문입니다. 친구들이
폰/콘솔/윈도우10+를 쓴다면 **베드락**이 나을 수 있습니다. 요소들이 어떻게
대응되는지, 무엇이 달라지는지 정리합니다.

## 베드락을 고를 때

- 친구들이 **모바일 / 콘솔 / 윈도우(베드락) 에디션**을 쓸 때.
- 파이 부하를 최대한 가볍게 하고 싶을 때(베드락 서버가 더 가벼움).

## 여기 자바 세팅과의 트레이드오프

| 항목 | 자바(이 저장소) | 베드락 |
|---|---|---|
| 파이 성능 | 튜닝하면 양호 | 대체로 더 가벼움 |
| **RCON** | ✅ 내장 | ❌ 미지원 |
| 원격 명령 | RCON(봇/CLI) | stdin/래퍼 또는 애드온 필요 |
| 치트/op 모델 | op 레벨(깔끔) | `permissions.json` + `server.properties`의 `allow-cheats` |
| 플러그인 | Paper 플러그인 | 비헤이비어 팩 / 제한적 서버 애드온 |
| 크로스플레이 | 자바만(또는 Geyser) | 베드락 기기 네이티브 |

## 파이에서 베드락 올리기

공식 Bedrock Dedicated Server(BDS)는 **x86 전용**이라, ARM에서는 다음 중 하나를
씁니다:

- **[PocketMine-MP](https://pmmp.io/)**(PHP) — ARM 네이티브, 플러그인 지원,
  구동 가능한 콘솔 제공.
- **`box64`로 BDS**(x86 에뮬레이션) — 되지만 오버헤드가 큼; 파이엔 부적합.

ARM에선 보통 PocketMine를 씁니다.

## 베드락에서 소유자만 치트

베드락 모델은 자바와 다릅니다:

- `server.properties`에 `allow-cheats`와 `default-player-permission-level`이 있음.
- `default-player-permission-level=member`로 두어 일반 플레이어가 명령을 못 쓰게
  하고, `permissions.json`에서 XUID로 나에게 **operator**를 부여.
- 서버 콘솔(및 PocketMine 콘솔/플러그인)은 항상 명령을 실행할 수 있음 — 그게 내
  원격 치트 채널.

## RCON 없이 원격 관리

베드락은 RCON이 없으므로 원격 관리 방식을 바꿔야 합니다:

- **PocketMine**: 콘솔 브리지 플러그인이나 커맨드 API 사용; 디스코드 봇이 RCON
  대신 플러그인의 로컬 소켓/HTTP와 통신.
- **BDS**: 서버를 감싸 **stdin**에 쓸 수 있게 하고(예: `tmux`/`screen` 세션이나
  작은 슈퍼바이저), 봇이 거기로 명령을 보냄.

이 저장소의 디스코드 봇은 RCON 기반입니다. 베드락을 지원하려면 `bot/rcon.py`를
stdin/플러그인 전송으로 바꾸고 나머지(관리자 게이팅, 로딩 애니메이션, 로그)는
그대로 두면 됩니다.

## 크로스플레이 옵션(자바 유지 + 베드락 합류)

자바 세팅이 마음에 들지만 베드락 친구가 있다면, Paper 플러그인으로
**[Geyser](https://geysermc.org/)**(+ Floodgate)를 추가하세요. 그러면 베드락
클라이언트가 자바 서버에 접속합니다. RCON·ops·이 저장소의 모든 것이 유지되어,
혼합 친구 그룹엔 대개 최선입니다.
